# Минимальный legal‑чек‑лист для сайтов (РБ)

Дисклеймер: это **не юридическая консультация** и не заменяет юриста. Это практичный чек‑лист для онбординга, чтобы не упереться в “нечего выкатывать” на релизе.

Цель: понять, какие юридические элементы **нужны** для конкретного сайта, и **кто** их предоставляет/утверждает.

## 0) Вопросы на 5 минут (сразу на онбординге)

- [ ] Кто юридическое лицо/ИП, от чьего имени сайт? (реквизиты)
- [ ] Будут ли формы/заявки/обратная связь (сбор телефона/e‑mail/имени)?
- [ ] Будут ли онлайн‑платежи/заказы/доставка/возвраты?
- [ ] Кто утверждает юридические тексты? (decision maker / бухгалтер / юрист)
- [ ] Нужна ли мультиязычность и какие версии будут юридически “главными”?

## 1) Реквизиты и идентификация

- [ ] На сайте есть страница/блок “Контакты” с актуальными контактами
- [ ] Если применимо: реквизиты организации/ИП (что именно — уточняет клиент/бухгалтер)
- [ ] Публичные способы связи (e‑mail/телефон/адрес) согласованы

## 2) Персональные данные (если есть формы/заявки)

Если сайт собирает хотя бы e‑mail/телефон — это уже чувствительная зона.

- [ ] Понимаем, какие данные собираем (имя/телефон/e‑mail/сообщение/файлы)
- [ ] Понимаем, зачем собираем (связаться, расчёт, консультация)
- [ ] Понимаем, куда попадают данные (почта, CRM, таблица, мессенджер)
- [ ] Есть текст “согласия”/уведомления у формы (кто предоставляет — клиент)
- [ ] Есть ссылка на политику/условия обработки (если клиент готовит)

Примечание: если клиент говорит “нам не надо про ПДн” — всё равно уточните маршрут данных (почта/CRM), чтобы не было сюрпризов.

## 3) Cookies / аналитика (если ставим метрики)

- [ ] Какие системы аналитики используем (например, GA/метрика/другие)
- [ ] Есть ли cookies/трекинг и нужно ли уведомление/баннер (решение: клиент)
- [ ] Кто отвечает за тексты и юридическую часть уведомления (клиент/юрист)

## 4) Условия/оферта/возвраты (если есть продажи/заказы)

Если на сайте есть оформление заказа, оплата, доставка:
- [ ] Есть/будут условия продажи/оферта (клиент предоставляет текст)
- [ ] Есть политика возвратов/отмены (если применимо)
- [ ] Понятно, как пользователь принимает условия (чекбокс/ссылка/текст)

## 5) Интеллектуальная собственность и права на контент

- [ ] Кто владеет текстами/фото/видео/иконками (клиент/подрядчик)
- [ ] Есть подтверждение прав на материалы (особенно фото/видео из стоков)
- [ ] Если используете сторонние ассеты/шрифты — лицензии ОК

## 6) Ответственность и роли (внутри онбординга)

Зафиксируйте в `Risks, Assumptions & Exclusions`:
- [ ] кто предоставляет юридические тексты (клиент)
- [ ] кто утверждает и несёт ответственность за юридическую корректность (клиент)
- [ ] вы внедряете тексты/ссылки/чекбоксы “как предоставлено”

## 7) Минимальный набор “что попросить у клиента”

- [ ] Реквизиты (что публиковать) или подтверждение “публикуем только контакты”
- [ ] Текст политики/условий (если требуется)
- [ ] Формулировка согласия для форм
- [ ] Текст для cookies/аналитики (если требуется)
- [ ] Тексты про оплату/доставку/возвраты (если есть продажи)
